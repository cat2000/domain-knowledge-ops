# 下单与改单 · 领域知识定稿

> English SSOT: [`ordering-domain-brief.md`](./ordering-domain-brief.md)

> **Fixture only** — fictional Acme Orders. Not production tenant data.

## 概述与范围

- **适用对象**：买方采购员（purchaser）、只读观察员  
- **范围**：报价有效期内对 **Open** 订单的行数量修改；履约后只读规则  
- **不在本文**：定价重谈、加 SKU、WMS 内部拣货

## 不在本文展开

- 仓储算法、内部值班表、纯 API 字段对账表

## 领域模型摘要

- **一等业务对象**
  - **订单（Order）** — 买方可见的订购承诺
  - **报价版本（Quote Version）** — 决定可否按下单/改单价与有效期
  - **订单行（Order Line）** — 可改数量的最小单位（未发货时）
- **对象关系**
  - 订单 **绑定** 一个报价版本
  - 订单 **包含** 多个订单行
- **状态机/状态转换**
  - Open → Partially Shipped → Shipped；任一时刻可 → Cancelled

## 核心业务规则

### 规则簇 · Open 且报价有效时可改数量

- **已确认规则**
  - **适用对象**：采购员角色
  - **条件**：订单状态为 `Open`；绑定的 `quote_version` 未过期
  - **处理分支**：允许修改未发货行的数量并保存；保存成功后订单明细展示新数量与新合计
  - **时间窗**：以报价有效期截止时刻为准
  - **例外**：无（本簇）
  - **用户可见后果**：采购员看到可编辑数量控件；保存后 3 秒内明细合计更新
- **待确认边界**
  - 数量上调是否需卖方审批（工单未决）
- **用户可见影响**
  - 可改 vs 禁用保存并提示需新报价
- **关联待确认事项**
  - 见「待确认事项」· 数量上调审批

### 规则簇 · 报价过期不可保存改单

- **已确认规则**
  - **适用对象**：采购员
  - **条件**：`quote_version` 已过期
  - **处理分支**：禁用 Save；展示需新报价的提示
  - **时间窗**：过期后立即生效
  - **例外**：无
  - **用户可见后果**：无法保存改单；看到过期提示
- **待确认边界**
  - 提示文案具体用词（工单 TBD）
- **用户可见影响**
  - 阻断改单保存
- **关联待确认事项**
  - 过期横幅文案

### 规则簇 · 已发货行只读

- **已确认规则**
  - **适用对象**：采购员
  - **条件**：订单为 `Partially Shipped` 或行已发货
  - **处理分支**：已发货行只读；未发货行仍可按 Open/有效报价规则编辑
  - **时间窗**：无特定时间窗
  - **例外**：整单 `Shipped` 或 `Cancelled` 时全部改单控件隐藏
  - **用户可见后果**：部分行可编、部分行只读，或整单无改单入口
- **待确认边界**
  - Partially Shipped 后是否一律需卖方审批（工单开放问题）
- **用户可见影响**
  - 控件显隐与只读
- **关联待确认事项**
  - 部分发货审批

## 术语说明

- **报价版本（Quote Version）** — 合同价与有效期的版本标识；过期后不可按原价改单保存
- **Open** — 尚未进入发货、仍可改数量的订单状态（本定稿口径）

## 待确认事项

- **影响规则**：Open 且报价有效时可改数量
  - **待确认/待补充**：数量上调是否需要卖方审批
  - **建议确认人**：PO / 卖方运营
  - **确认后影响**：可能增加审批分支或保持采购员自助
- **影响规则**：报价过期不可保存改单
  - **待确认/待补充**：过期横幅最终文案
  - **建议确认人**：UX / PO
  - **确认后影响**：仅文案，不改分支
- **影响规则**：已发货行只读
  - **待确认/待补充**：Partially Shipped 是否强制卖方审批
  - **建议确认人**：PO
  - **确认后影响**：可能新增审批状态
