## 本质

报告是 **sprint 决策输入**，优化目标是 **决策延迟**。

## 铁律 P1–P9

| # | 铁律 | 原因 |
|---|------|------|
| P1 | **摘要必先答「做什么」**：用 **范围** 小节写用户可见能力/页面/边界（见 P10），再列须先拍板项 | 读者不应从 R 条反推 scope |
| P2 | **用户可见标题用中文决策语**：`## 摘要`（非 Layer 1）；`### 完整风险清单（FULL_UNKNOWN_MAP）` 等 — 英文键仅作括号副标 | 降低产品/业务阅读门槛 |
| P3 | **条目标题保留 `#### R-00N · 严重度 · 主题`**（门禁脚本依赖），但主题用 **白话**（如「合同四页还是五页」） | 可追溯 + 可扫读 |
| P4 | **冲突/口径不一致**（工单 vs 设计 vs 定稿）→ 在 `证据` 下附 **≤4 列窄表**：`来源 \| 说法 \| 待裁决` | 对照比长段落更快决策 |
| P5 | **`利害` 最多 2 条短 bullet**：用户可见后果 / 交付或流程后果；禁止一句塞三个从句 | 降低解码成本 |
| P6 | **必须修复项**在 `建议（D）` 或 `补充` 加 **`建议确认人`**（产品/合规/后端/测试等角色，不编造姓名） | 会议可直接指派 |
| P7 | **定稿已写清的边界** → 降级应当澄清或标「与领域库一致」；**仅冲突**标 `[DOMAIN_KNOWLEDGE]` | 避免对已知规则重复报 MUST |
| P8 | **`brief` 模式**：只出 `## 摘要` + `EVIDENCE_COVERAGE`；摘要仍含 **范围**、须先拍板、计数、就绪度 | 管理层/站会短读 |
| P9 | **禁止 meta 自夸**（「易读版」「为便于阅读」）；禁止向非技术读者解释 Layer/R/D 含义 | 结构自说明 |

**P10**：见 [`../../_shared/presentation-p10.md`](../../_shared/presentation-p10.md)。

## stage → 呈现深度

| stage | 摘要 | Layer 2 |
|-------|------|---------|
| `intake` | 必须；Top MUST ≤5 | 可省略完整风险清单；保留 `EVIDENCE_COVERAGE` |
| `refinement`（默认） | 必须 | 完整风险清单 + 验收可测点 |
| `pre_sprint` | 必须 | 完整 + 必须修复项尽量带确认人 + 治理字段 |

## 推荐摘要结构

```markdown
## 摘要

**范围**：……（一句，写能力/页面/不做什么）

**一句话**：……

**须先拍板**：……

**计数**：必须修复 N · 应当澄清 N · 可选 N
**就绪度**：……
```

（计数分隔符用 `·` 或 `/` 均可；须与 R 标题对账。）
