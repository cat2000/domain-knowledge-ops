## 本质

分解项是 **迭代可执行承诺**；读者要先知道「切了几块、建议顺序、每块怎么验」，再读字段。

**标题本质**：标题是该项承诺的 **3 秒头条**（规划会扫一眼就知道「交付什么结果」），**不是**把 `scope` 里多条改动压成顿号清单。一项可含多个改动；标题只挂 **一个主结果**，其余进 `scope` / `done_when`。

**scope 本质**：`scope` 每条 bullet 是 **一个可口述的关注点**（一块控件/一条用户可见行为/一条边界），**不是**把 PRD 字段表用冒号、分号、斜杠焊成一句「覆盖清单」。

## 铁律 P1–P12

| # | 铁律 | 原因 |
|---|------|------|
| P1 | 输出开头 **范围** 一句（与 risk 同口径；直接写做什么，不用「本票范围」） | 先边界后条目 |
| P2 | 每项 **验收标准（done_when）** 写可观测行为，禁止「开发完成」 | INVEST/Testable |
| P3 | 源文削弱可测性 → **纠偏说明** 置首，再列 Spike/Story/Enabler | 先纠偏后拆单 |
| P4 | 无 preamble；不向读者解释 ticket_system 规则本身 | 降噪声 |
| P5 | 范围后加 **`## 拆单一览`**（3–5 行）：共几项、建议迭代顺序、是否含阻塞 Spike、可选 Enabler 标注 | 规划会 30 秒读懂 |
| P6 | **纠偏说明用叙事段落**；(a)(b)(c) 仅作 Agent 内心结构，**默认不印出字母标签** | 降 INVEST 术语门槛 |
| P7 | 若同会话已跑 `@requirement-risk` 或存在未决冲突，Spike/Story **注明对应 R-00N**（如「对应 risk R-001」） | risk→split 可追溯 |

| P9 | 条目标签：**中文小标题 + 英文键括号**（见下表） | 降双语扫描成本 |
| P11 | **标题一口可读**：一个主结果；禁顿号/「与」堆三条改动；禁空动词与私货缩写（见下） | 标题=承诺头条，不是 scope 摘要 |
| P12 | **scope 一条一意**：一条 bullet 一个关注点；禁「覆盖 X：A；B / C / D」式焊接（见下） | scope=可口述边界，不是字段表压缩 |

## 标题一口可读（P11）

**自检（不过则改标题，不先拆项）**： refinement 上能 **一口气读完并复述主结果**；听者不必先拆「A、B 与 C」。

| 要 | 不要 |
|----|------|
| 一个主结果（对象 + 可观察结果） | 用 `、` / `与` 把三项改动并列进标题 |
| 附带改动写进 `scope`，或用破折号/冒号挂一句从属说明 | 把 `scope` 三条 bullet 压缩进标题 |
| 口语完整词：直接下载、弹窗、去掉模板页 | 私货缩写：直下、开能力、落地改造 |
| 实义动词：改为 / 启用 / 去掉 / 支持用户… | 空动词：开放、赋能、完成（无对象结果） |

**反例 → 正例**

- 反：`开放 DBL 申请须知页的对公合同下载、营业执照须知弹窗与合同直下`
- 正：`DBL 申请须知页可直接下载对公与纸质合同，营业执照须知改为弹窗`  
  （主结果=须知页下载可达；执照弹窗为同页从属说明；「去掉模板页」只写 `scope`）

## scope 一条一意（P12）

**自检（不过则拆 bullet，不先拆 Story）**： refinement 上能 **一口气读完并复述这一条在说什么**；听者不必在半句里切换「角标 / 筛选 / 数量」多个话题。

| 要 | 不要 |
|----|------|
| 一条 bullet = 一个关注点（角标、筛选、卡片字段、取消提示…分条写） | `覆盖 X：A；B / C / D` 把多控件焊进一句 |
| 列表枚举可单独成条，或写成「可按甲、乙、丙筛选」整句 | 用 `/` 串一长串筛选项还夹「红点+数量」注解 |
| 卡片字段多时分条或「卡片展示：…」后用短句，勿分号连压三主题 | `卡片：…；提示 A；提示 B` 三主题一分号焊死 |

**反例 → 正例**

- 反：`覆盖「我的活动」Tab：未填全时 Tab「待完善」角标；筛选全部 / 待完善（红点+数量）/ 进行中（数量）/ 已取消（数量）`
- 正：
  - `有报名信息未填全时，「我的活动」Tab 显示「待完善」角标`
  - `列表可按全部、待完善、进行中、已取消筛选；待完善显示红点与数量，进行中显示数量`

## 字段标签（P9）

| 中文标签 | 英文键（rule） |
|----------|----------------|
| 标题 | title |
| 范围 | scope |
| 验收标准 | done_when |
| 依赖 | depends_on |
| 信心 | confidence |

**条目类型标题**：`Spike N` / `用户故事 N（User Story）` / `技术任务 N（Tech Task · Enabler · 可选）`

## 推荐输出结构

```markdown
**范围**：……

## 拆单一览
- 共 N 项：Spike 1 → 用户故事 1 → …
- 阻塞：……
- 可选：技术任务 1（Enabler）仅在后端联调需要时纳入

**纠偏说明**（仅 INVEST 需纠偏时；叙事段，无 (a)(b)(c) 标签）
……

Spike 1 / 用户故事 1 / …
```

**`brief` 模式（P8）**：只出 **范围 + 拆单一览 + 纠偏说明（若有）**；不展开各条 scope/done_when 明细；门禁用 `--brief`。
